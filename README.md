# Healthcare-Workflow-Improvement-Case-Study

## Project Overview

This case study demonstrates an operational analysis of a fictional
hospital echocardiography department.

The project focuses on identifying workflow bottlenecks related to:

- Patient wait times
- Scheduling
- Prior authorization
- Room utilization
- Staffing capacity
- Billing workflow

The dataset used in this repository is entirely synthetic and was
created for portfolio demonstration purposes.

## Business Problem

The department is experiencing inconsistent patient wait times,
delays between referral and appointment completion, and uneven
utilization of available resources.

The goal of this analysis is to answer:

1. Where are the largest delays in the workflow?
2. Which days and times have the highest demand?
3. How efficiently are available rooms being utilized?
4. Where do authorization delays affect scheduling?
5. What operational changes could improve patient throughput?

## Key Findings

- Authorization delays contributed to longer referral-to-scheduling times.
- Demand was concentrated during specific weekday and hourly periods.
- Room utilization was inconsistent across the operating schedule.
- Certain workflow handoffs created unnecessary administrative delays.

## Recommendations

- Implement authorization status tracking.
- Align scheduling capacity with peak demand.
- Standardize handoff procedures.
- Monitor room utilization and patient wait times through a management dashboard.
